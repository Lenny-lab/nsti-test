# Web 调研笔记（截至 2026-04-14）

## 1. 现在网上说的 MBTI / STi，大多在说什么

- 中文互联网里常见的 “STi/sti 测试” 并不是一个统一、标准化的学术量表名，更常见的是对“类型化人格测试”的泛称。
- 热度最高的底层叙事仍然是 MBTI：基于荣格类型学，用 4 个偏好维度组合出 16 种类型。
- 大众传播里还常把 MBTI、16Personalities、荣格八维、小测试平台上的校园版/恋爱版/宿舍版/专业版混着叫，这也是 “各种 STi 都很火” 的来源之一。

## 2. 背后的常见逻辑

### 2.1 经典 MBTI 逻辑

- 4 个二分维度：E/I、S/N、T/F、J/P。
- 题目通常是自陈式：让用户判断“更像我”还是“更不像我”。
- 实际上大多数题目都在测连续谱，但结果展示为了传播方便，经常会被压缩成四个字母标签。

### 2.2 16Personalities / 网络人格测试逻辑

- 依旧沿用 16 型输出形式，方便用户理解、社交传播和“代入”。
- 题目设计和结果呈现更偏大众化产品：强调故事感、标签感、分享感。
- 很多测试会在 MBTI 四字母框架上额外加入强度百分比、A/T 之类的辅助标签。

### 2.3 校园版 / 地域版 / 场景版 STi 的逻辑

- 本质上是在“维度 + 标签 + 情境化题干”这个框架上做二创。
- 变化最大的不一定是计分方式，而是题目语言和结果文案：
  - 把抽象人格表述改成校园可感知行为。
  - 把结果页从“你是什么人”改成“你在学校里更偏好怎样学习、合作、社交、推进事情”。
- 所以做 NNU 版时，真正关键的不是模仿 MBTI 的字母，而是把题干、维度命名、结果解读做成南师大语境下可讨论、可传播的版本。

## 3. 为什么这类内容会火

- 低门槛：做题成本低，立刻拿到一个“身份标签”。
- 易传播：四字母或几个维度很适合做头像、签名、社交卡片和宿舍讨论。
- 易二创：可以很快衍生出“学院版/宿舍版/恋爱版/食堂版/城市版”。
- 易共鸣：题目常使用温和、模糊、普适的表述，用户容易感觉“说得像我”。

## 4. 对 NSTI 有什么直接启发

结合你项目里现有的 4 轴设计，外部调研后我认为目前方向是对的，尤其适合继续强化以下几点：

- 继续坚持“偏好画像”而不是“人格定性”。
- 题目尽量使用南师大校园内真实可感的行为场景：
  - 班群/课代表/社团招新/选修课小组/DDL/自习室/图书馆/活动统筹/展示汇报。
- 结果页除了 4 字母，更要展示每条轴的强度。
- 类型文案不要写成“优劣排名”，要写成“更适合什么场景、容易踩什么坑、和谁搭更舒服”。
- 采集题库时，不要只盯着 MBTI 原题，更要大量收集：
  - 学习风格题
  - 协作偏好题
  - 社交能量题
  - 时间管理题
  - 决策偏好题

## 5. 本次外部检索得到的可用来源类型

### 5.1 权威背景源

- Myers & Briggs Foundation：MBTI 基本理论与偏好维度说明。
- 16Personalities：大众版 16 型测试与产品化表达方式。

### 5.2 可直接抽题的公开网页

- 公开问卷页（尤其是问卷星公开页）。
- 公开测试站点（如 MBTI 93 题、荣格八维前端测试页）。
- 一部分文章页会列出题目逻辑、题型结构和维度映射。

### 5.3 社交平台线索源

- 微博：能搜到 MBTI/STi 账号、帖子、话题页，但很多内容是图文卡片，直接抽题经常要 OCR。
- 小红书：大量相关内容存在，但公开搜索、登录态、反爬和前端渲染限制更重。
- 微信公众号：文章页可作为线索源，但公开搜索入口有限，很多内容需要先靠搜索引擎或第三方搜索发现。

上面这三类平台更适合作为“线索发现 + 人工筛选 + 定向抓取”的来源，不适合一开始就当成稳定、统一的公开题库接口。

## 6. 当前建议的策略

先把数据工作流拆成两层：

1. 先高强度抓公开网页题库，快速形成可落库的 CSV。
2. 再把微博 / 小红书 / 公众号做成“受限平台线索层”，按关键词、链接、截图、图片 OCR 去补充。

这会比一上来就硬怼三大平台爬虫更快拿到结果，也更符合你当前原型迭代需要。
